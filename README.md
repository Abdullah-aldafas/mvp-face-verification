# تحقق ميداني - Absher Tuwaiq Hackathon

### نبذة عن الفكرة
"تحقق ميداني" هو نموذج أولي (MVP) لنظام يساعد رجال الأمن في الميدان
على التحقق من هوية الأشخاص عن طريق **تصوير الوجه** باستخدام الجوال،
ثم عرض حالة الهوية/الإقامة والمخالفات الأساسية من جهات مختلفة
(مثل الجوازات والمرور والأحوال المدنية) في **شاشة واحدة بسيطة**.

### المشكلة التي نحلّها
- التحقق الميداني حاليًا يعتمد على أنظمة متفرقة وإدخال يدوي.
- هذا يسبب بطء في الحملات الميدانية ويزيد احتمالية الأخطاء.
- رجل الأمن يحتاج واجهة واحدة سريعة ومصممة لبيئة الميدان.

### الحل المقترح
- تطبيق مخصص لرجال الأمن:
  - تصوير وجه الشخص في الموقع.
  - مطابقة الصورة مع سجلات تجريبية (Mock Data).
  - عرض ملخص واضح:
    - الاسم
    - رقم الهوية / الإقامة
    - حالة الهوية (سارية، منتهية، عليها إيقاف)
    - جهة العمل (للمقيمين)
    - عدد المخالفات المرورية النشطة
    - وجود ملاحظات أمنية إن وجدت

### مكونات الـ MVP الحالية
- **تصميم UX/UI في Figma** يتضمن:
  - شاشة ترحيب / دخول
  - شاشة "تحقق جديد" لالتقاط أو رفع صورة الوجه
  - شاشة تحميل "جاري التحقق من الهوية..."
  - شاشة نتيجة التحقق (بطاقة معلومات الشخص)
  - شاشة لوحة المتابعة (Dashboard) لإحصائيات التحقق
- النموذج الحالي يركّز على تجربة الاستخدام، مع قابلية
  تطوير نسخة برمجية لاحقًا (Backend + Face Recognition).

### أفكار للتطوير المستقبلي
- ربط فعلي مع أنظمة الجوازات والمرور والأحوال عبر واجهات APIs.
- استخدام نموذج حقيقي للتعرّف على الوجه (Face Recognition).
- دعم أنواع تحقّق أخرى (رقم الهوية، رقم الإقامة، رقم لوحة المركبة).
- إضافة صلاحيات مختلفة للجهات والرتب داخل التطبيق.
